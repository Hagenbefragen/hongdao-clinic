# Start with Why — Golden Circle Framework

## Purpose

This skill transforms product messaging from feature-first (WHAT) to purpose-first (WHY) using Simon Sinek's Golden Circle methodology. It ensures every landing page, pitch deck, blog post, and investor narrative starts from the **limbic brain** (emotion, trust, loyalty) before engaging the **neocortex** (rationality, features, specs).

> **"People don't buy WHAT you do; they buy WHY you do it."** — Simon Sinek

## Narrative Construction Rules

### Rule 1: Start with WHY, Always
Never open with "We have 108 agents." Open with "The planet is on fire and your AI is making it worse."

### Rule 2: Use Rosetta Stone Words
Map every technical feature to a word the audience already knows (Firewall, DLP, SIEM, Zero Trust, Compliance-as-Code).

### Rule 3: Three-Layer Emotional Cascade
1. **FEAR** → The regulation tsunami, the €35M fine, the insurance premium doubling
2. **HOPE** → A world where AI safety is green, sovereign, and affordable
3. **PROOF** → 8.06M prompts tested, 95.49% TPR, 0.46ms latency, 12σ quality target

### Rule 4: The Biological Advantage
The Golden Circle maps to brain biology:
- **WHY** → Limbic brain (feelings, trust, loyalty, decision-making)
- **HOW/WHAT** → Neocortex (rational thought, language, analytical thinking)

People make decisions with the limbic brain, then rationalize with the neocortex. Start with feelings, end with facts.

### Rule 5: The CO₂ Credit Narrative
Destill.ai is not only an AI Safety company — we can prove CO₂ savings with metrological data:
- Every blocked adversarial prompt saves GPU cycles
- Saved GPU cycles = measurable kWh = provable CO₂ reduction
- Provable CO₂ reduction = tradeable CO₂ credits
- **We are an AI Safety company that generates environmental credits as a byproduct**

## Application Checklist

When building any customer-facing content, verify:

- [ ] Does the FIRST thing the audience sees answer "WHY does this exist?"
- [ ] Is the "regulation tsunami" urgency established within 3 seconds?
- [ ] Are Rosetta Stone words used (Firewall, DLP, SIEM, etc.)?
- [ ] Is the CO₂/energy narrative integrated (not just safety)?
- [ ] Does the insurance angle appear (NI-Shield, premium reduction)?
- [ ] Are the three product clusters clearly mapped to WHY?
- [ ] Is there visceral imagery (power plants, heat, GPU waste)?
- [ ] Does the page end with proof (benchmarks, numbers, evidence)?
- [ ] LLM Agnostic ✅ — works with any LLM
- [ ] Device Agnostic ✅ — CPU, NPU, laptop, VPS, edge
- [ ] No GPU dependency ✅ — sovereign, green
