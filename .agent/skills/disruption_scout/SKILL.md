# 🌊 Disruptive Innovation Scout

> "The future is already here — it's just not evenly distributed." — William Gibson

## Purpose

The Disruptive Innovation Scout is OHM's **foresight weapon**. While the Horizon Scanner looks at broad trends (H3), this skill goes deeper — it **actively combines technologies, identifies invention whitespace, and generates patentable innovation hypotheses** that keep OHM 2-5 years ahead of the market.

- **Monthly**: Run a full disruption scan to identify new convergences

- **During patent sessions**: Generate novel combinations from existing OHM IP
- **After competitor announcements**: Assess threat level and counter-innovate

### Layer 2: Christensen Disruption Test

For each potential innovation, apply Clayton Christensen's framework:

1. **Is it simpler?** Does it remove complexity that incumbents impose?
2. **Is it cheaper?** Can it serve non-consumers at lower cost?
3. **Is it more convenient?** Does it work where/when incumbents don't?
4. **Does it create a new value network?** Does it redefine who benefits?
5. **Will incumbents dismiss it?** Is it "not good enough" for their market?

Score: 5/5 = **Disruptive** | 3-4/5 = **Sustaining** | <3 = **Incremental**

## Execution Protocol

When triggered, the agent SHALL:

1. **SCAN**: Read all existing patents in `Documentation/Patents/`
2. **MAP**: Cross-reference with the 6 technology domains above
3. **IDENTIFY**: Find 3-5 convergence points not yet exploited
4. **COMBINE**: Generate 5-10 novel combinations using the engine
5. **TEST**: Apply Christensen's 5-point disruption test to each
6. **SCORE**: Rate each combination on disruption potential (1-10)
7. **PATENT**: For combinations scoring ≥8, draft a provisional patent abstract
8. **REPORT**: Output a "Disruption Radar" document with actionable next steps
