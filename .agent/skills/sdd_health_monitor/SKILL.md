# 🔬 SDD Health Monitor — "The System That Watches Itself"

> _Petal 6 of the Flower of Life SDD — MEASURE_ > _"What you cannot measure, you cannot improve." — Lord Kelvin_ > _"What you CAN measure but DON'T, will bite you." — OHM Ahnung_

## Purpose

The SDD Health Monitor is the **self-observing intelligence** of the Skill-Driven Development methodology. It answers the question every mature system must ask: _"How well am I performing, and am I getting better?"_

Without this skill, the SDD is a flower with only 5 petals — geometrically incomplete, unable to close its feedback loop. With it, MEASURE feeds back through Quantum Intuition (the Circle) into SENSE, completing the sacred geometry.

## When to Trigger

- After every major development session (weekly minimum)
- Before investor/customer demos (to show self-improvement evidence)
- When routing accuracy drops (Ahnung seems "off")
- After adding new skills (to detect integration health)
- During `/bestpractice` audits (as the measurement dimension)

## Core Metrics — The SDD Health Score

### 7 Dimensions of SDD Health

| #   | Metric                     | How Measured                                | Target             | Weight |
| --- | -------------------------- | ------------------------------------------- | ------------------ | ------ |
| 1   | **Skill Invocation Rate**  | Unique skills used per week                 | ≥30/56 (54%)       | 15%    |
| 2   | **Routing Accuracy**       | User correction rate vs. Ahnung auto-select | ≥85% no-correction | 20%    |
| 3   | **Pipeline Yield**         | Novel dissolutions per innovation run       | ≥1 per run         | 15%    |
| 4   | **PMF Hit Rate**           | market_fit_validator scores ≥6/10           | ≥70% of runs       | 10%    |
| 5   | **Workflow Completion**    | Triggered workflows that complete fully     | ≥80%               | 15%    |
| 6   | **Patent Generation**      | Claims drafted per innovation cycle         | ≥2 claims          | 10%    |
| 7   | **φ-Learning Convergence** | Fibonacci weight stability after N runs     | Stable by run ~13  | 15%    |

### Composite Score Calculation

```
SDD Health = Σ (dimension_score × weight) / Σ weights

Thresholds:
  90-100%  → 🌸 BLOOMING    (Flower of Life fully open)
  75-89%   → 🌿 GROWING     (Healthy growth trajectory)
  60-74%   → 🌱 SPROUTING   (Establishing roots)
  40-59%   → 🌰 SEEDING     (Early stage, needs attention)
  <40%     → ⚠️ DORMANT     (System not being used effectively)
```

## Data Collection Protocol

### Automated Sources

```markdown
1. **Git commit messages** → Count skill mentions (e.g., "feat(ship_engine):")
2. **Workflow trigger logs** → Track /workflow invocation frequency
3. **Innovation reports** → Extract PMF scores, dissolution counts
4. **Patent documents** → Count new claims per cycle
5. **KI metadata** → Track knowledge item creation/update frequency
```

### Manual Input (Lightweight)

At the end of each development session, answer 3 questions:

1. Did Ahnung suggest the right skills? (Y/N/Partial)
2. How many distinct workflows did you trigger? (Number)
3. Did you discover something genuinely novel? (Y/N)
