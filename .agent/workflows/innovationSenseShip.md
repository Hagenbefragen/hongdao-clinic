Map the dissolution against applicable regulations:

| Regulation | Check                                      | Competitive Angle              |
| ---------- | ------------------------------------------ | ------------------------------ |

| eIDAS 2.0  | Digital identity requirements              | "SSI-native"                   |
| NIS2       | Cybersecurity obligations                  | "Sovereign security"           |
| MiCA       | Crypto/token compliance (if XOM involved)  | "Regulated economy"            |
| DORA       | Operational resilience (if financial)      | "Offline-first resilience"     |
| GDPR/DSGVO | Data processing basis                      | "Edge-first, no PII on server" |

**Key principle:** Turn compliance into competitive advantage. Every regulation we satisfy that competitors don't = selling point.

Read `regulatory_navigator/SKILL.md` for compliance-to-feature mapping template.

## Step 10c: `quality_gateway` ✅

Final quality gate — 5 dimensions:

| Gate           | Question                            | Pass if...              |
| -------------- | ----------------------------------- | ----------------------- |
| Consistency    | Contradicts existing docs/KIs?      | No contradictions found |
| Plausibility   | Are claimed improvements realistic? | Evidence-backed numbers |
| OHM Alignment  | Sovereignty, privacy, edge-first?   | Serves the mission      |
| Transparency   | Simulated vs real clearly labeled?  | No hidden assumptions   |
| Persuasiveness | Would a skeptical CTO accept?       | Data + logic + demo     |

**Requirement:** 4/5 PASS minimum.
If < 4 PASS → route failures back to originating skill for correction.

Read `quality_gateway/SKILL.md` for scoring weights and re-routing rules.
